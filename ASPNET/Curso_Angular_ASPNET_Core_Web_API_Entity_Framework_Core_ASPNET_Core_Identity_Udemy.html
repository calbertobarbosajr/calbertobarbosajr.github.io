Visão Geral

O projeto é um sistema completo de gerenciamento de eventos, utilizando uma API de maturidade nível 3 — o mais alto nível de maturidade para APIs REST, conforme ensinado no curso.

Inicialmente, o sistema foi construído utilizando .NET 5 e SQLite. No entanto, estou atualizando gradualmente para:

  • ✔ .NET 9 (versão mais recente do framework);
  • ✔ PostgreSQL como banco de dados, substituindo o SQLite.

Essa atualização não é apenas técnica, mas também estratégica:

Optei por usar uma tecnologia diferente do curso para evitar apenas replicar o código do instrutor (copia e cola). Além disso, o uso da combinação ASP.NET + PostgreSQL, menos comum que ASP.NET + SQL Server

A infraestrutura do PostgreSQL foi configurada utilizando Docker, garantindo maior escalabilidade e facilidade de manutenção no ambiente de desenvolvimento.

Tecnologias Utilizadas

  • .NET 5 (em atualização para .NET 9)
  • EF Core
  • PostgreSQL (em substituição ao SQLite)
  • Angular
  • Reactive Forms
  • DTOs e Data Annotations
  • ASP.NET Core Identity para autenticação e autorização (JWT Token)
  • Swagger para documentação da API
  • Docker para gerenciamento do banco de dados PostgreSQL

Funcionalidades

  • Autenticação e autorização segura via JWT
  • Criação, atualização, consulta e exclusão de eventos
  • Paginação e filtros de pesquisa para otimizar consultas
  • Gestão de usuários e controle de acesso
  • Interface frontend responsiva utilizando Angular + Reactive Forms
